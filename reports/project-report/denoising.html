Simple Denoising (Matthias, 10pt)

This implementaion is based on the work by Pham et al. [1].

The groundturth is rendered using Path MIS, 1024 samples per pixel. It took 5 minutes. The image used for denoising is sampled with 16 samples per pixel and took roughly 20 seconds. Since the denoising is implemented inside nori and C++, it is fast. Denoising an image of this size (800x600) takes about one second (heavily dependent on the parameters).

Implementation

For the variance computation, please see Adaptive Sampling. Furthermore, I implemented equation 6 and 7 from [1]. This approach uses two factors in the computation of the new color: firstly the difference in pixels (g), where this is a gaussian kernel operating on the raw pixel distances. Secondly, it uses a kernel operating on the color values (f), which ensures that close by pixels should have close by colors. Both factors can be controlled by the corresponding sigmas (where sigma_d is for the distance and sigma_vr for the color distance). The denoiser is called from within the render thread (after it is done with rendering) and operators on the full image block.
