Bump Mapping or Normal Mapping (Roger, 5pt)

For normal maps we can add extra detail by perturbing the shading normal by a normal from a texture. The normal maps are encoded as png's where whole normal range [-1,1] per dimension is encoded in the color range [0,255]. This conversion is done once, when loading the image in PNGTexture.cpp. The sRGB conversion is disabled in the case of a normal map. There might be some small numerical issues within this conversion, so our results will not look exactly the same as the reference.
The normal map is stored in the shape base class, like the albedo texture in the diffuse bsdf. When setting the hit information, we apply the normal map. To apply this we use the `shFrame.toWorld` to convert the read normal perturbation from the image into world space. After some testing the normal map gave strange discontinuous results and I realised that the tangent must be aligned with the u coordinate. This means calculating smooth uv based tangents. Both of us worked on the calculating tangents seeing as it was also required for the disney anisotropic part.
To debug and validate I slightly extended the normal integrator to display the color of a relative direction to the normal. It simply shows the color of `shFrame.toWorld(direction)`. Using direction=(0,0,1) will display the normals. When changing this in the gui we can check that the colors remain smooth. Below we can validate that this method retains the normal when the identity normal map is used. The same is done for the bitangent=(0,1,0) and tangent(1,0,0).

An example render

We can see clearly that blender smoothly interpolates the values from the image, whereas our evaluation method does not. There are also differences in the uv mapping and thus the tangents. So the normal map looks like it is inverted.
